1. Модуль: Автентифікація (/api/v1/auth)
   Керує всіма процесами, пов'язаними з акаунтами користувачів.

Публічні ендпоінти
POST /register - Реєстрація нового користувача
Опис: Створює новий акаунт.

Тіло запиту (application/json):

JSON

{
"email": "test@example.com",
"password": "Password123!",
"firstName": "Іван",
"lastName": "Петренко",
"phone": "+380991234567"
}
Успішна відповідь (201 Created):

JSON

{
"success": true,
"message": "Реєстрація успішна. Будь ласка, підтвердіть email",
"user": { /_ ... об'єкт користувача ... _/ },
"tokens": {
"accessToken": "...",
"refreshToken": "...",
"expiresIn": 900
}
}
Помилки: 400 (невірні дані), 409 (email або телефон вже існує).

POST /login - Вхід в систему
Опис: Автентифікує користувача та видає токени доступу.

Тіло запиту (application/json):

JSON

{
"email": "test@example.com",
"password": "Password123!",
"rememberMe": false
}
Успішна відповідь (200 OK): Повертає об'єкт користувача та токени (аналогічно реєстрації).

Помилки: 400 (невірні дані), 401 (неправильний пароль або email).

POST /refresh - Оновлення токену доступу
Опис: Отримує нову пару токенів, використовуючи refreshToken.

Тіло запиту (application/json): refreshToken очікується в httpOnly cookie.

Успішна відповідь (200 OK):

JSON

{
"success": true,
"message": "Токен оновлено",
"tokens": {
"accessToken": "...",
"expiresIn": 900,
"tokenType": "Bearer"
}
}
Помилки: 401 (невалідний або відсутній refreshToken).

POST /logout - Вихід з системи
Опис: Видаляє сесію, пов'язану з refreshToken.

Тіло запиту: refreshToken очікується в httpOnly cookie.

Успішна відповідь (200 OK):

JSON

{
"success": true,
"message": "Успішно вийшли з системи"
}
POST /forgot-password - Запит на відновлення пароля
Тіло запиту (application/json):

JSON

{
"email": "test@example.com"
}
Успішна відповідь (200 OK):

JSON

{
"success": true,
"message": "Якщо email існує, ви отримаєте інструкції для відновлення пароля"
}
POST /reset-password - Встановлення нового пароля
Тіло запиту (application/json):

JSON

{
"token": "...",
"newPassword": "NewPassword123!",
"confirmPassword": "NewPassword123!"
}
Успішна відповідь (200 OK):

JSON

{
"success": true,
"message": "Пароль успішно змінено. Увійдіть з новим паролем."
}
Помилки: 400 (невалідний токен, паролі не співпадають).

Захищені ендпоінти (потребують accessToken)

DELETE /:id: Видалення товару.

POST /:id/images: Завантаження зображень.

POST /:id/variants: Створення варіанту товару.

Ні, це не всі. Попередній список містив основні ендпоінти для клієнтської частини (фронтенду).

Ось повний перелік, що включає адміністративні ендпоінти та ендпоінти для управління платежами, які також важливі для повноцінної роботи фронтенду, особливо для адмін-панелі.

Модуль: Адміністрування (/api/v1/admin)
Цей розділ призначений для керування магазином. Доступ вимагає ролі адміністратора.

1. Загальна інформація
   GET /dashboard: Отримує основну статистику для адмін-панелі (доходи, замовлення, клієнти).

GET /overview: Отримує загальний огляд стану системи.

POST /export: Запускає процес експорту даних (замовлення, товари, клієнти).

2. Керування користувачами (/users)
   GET /: Отримує список всіх користувачів з фільтрами.

PUT /:id/role: Змінює роль користувача (наприклад, з CUSTOMER на MANAGER).

PUT /:id/status: Активує або деактивує акаунт користувача.

POST /bulk: Дозволяє виконати масові дії над користувачами.

3. Керування товарами (/products)
   POST /: Створює новий товар.

PUT /:id: Оновлює існуючий товар за його ID.

DELETE /:id: Видаляє товар (м'яке видалення).

POST /:id/images: Завантажує зображення для товару.

POST /:id/variants: Створює новий варіант для товару (наприклад, інша вага або упаковка).

POST /import: Імпортує товари з файлу (CSV/Excel).

4. Керування категоріями (/categories)
   POST /: Створює нову категорію.

PUT /:id: Оновлює категорію.

DELETE /:id: Видаляє категорію.

POST /reorder: Змінює порядок сортування категорій.

5. Керування замовленнями (/orders)
   GET /: Отримує повний список замовлень з розширеними фільтрами.

GET /:id: Отримує детальну інформацію про конкретне замовлення.

PATCH /:id/status: Змінює статус замовлення (наприклад, "Обробляється" -> "Відправлено").

DELETE /:id/cancel: Скасовує замовлення.

6. Модерація відгуків (/reviews)
   GET /: Отримує список всіх відгуків для модерації.

PUT /:id/approve: Схвалює відгук, роблячи його видимим на сайті.

PUT /:id/reject: Відхиляє відгук.

POST /bulk: Дозволяє масово схвалити або відхилити відгуки.

7. Керування знижками (/discounts/admin)
   GET /admin: Отримує список всіх знижок та промокодів.

POST /admin: Створює нову знижку.

PUT /admin/:id: Оновлює існуючу знижку.

DELETE /admin/:id: Видаляє знижку.

GET /admin/:id/stats: Показує статистику використання знижки.

Модуль: Платежі (/api/v1/payments)
Цей модуль відповідає за обробку платежів.

POST /: Створює новий платіж для замовлення. Повертає URL для перенаправлення на сторінку оплати.

POST /:id/refund: (Адмін) Ініціює повернення коштів за платежем.

GET /:id/verify: Перевіряє поточний статус платежу.

POST /webhook/liqpay: Endpoint для отримання повідомлень від LiqPay про стан платежу (не для прямого виклику з фронтенду).

POST /webhook/monobank: Endpoint для отримання повідомлень від Monobank.

GET /payment/liqpay/:paymentId: Проміжний ендпоінт, який генерує HTML-форму і автоматично перенаправляє користувача на сторінку оплати LiqPay.

GET /payment/monobank/:paymentId: Аналогічно для Monobank.
