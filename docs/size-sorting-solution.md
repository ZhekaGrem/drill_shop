# Рішення для сортування розмірів варіантів

## Проблема

Варіанти продуктів (розміри) відображалися в невірному порядку:

- Літерні розміри: `L, XS, M, XXL` замість `XS, S, M, L, XL, XXL`
- Числові розміри: `48, 46, 47` замість `46, 47, 48`

## Рішення

### 1. Створена утиліта `size-sort.ts`

**Файл:** `src/shared/utils/size-sort.ts`

Функції:

- `sortSizes(sizes: string[])` - сортує масив розмірів
- `sortVariantsBySize(variants)` - сортує варіанти продуктів за розміром
- `compareSizes(a, b)` - порівнює два розміри

### 2. Підтримувані формати розмірів

#### Літерні розміри (в порядку):

- XXS, XS, S, M, L, XL, XXL, XXXL, 3XL, 4XL

#### Числові розміри:

- 46, 47, 48, 49, 50... (в числовому порядку)
- Підтримка діапазонів: 46-48, 50-52

### 3. Оновлені компоненти

Сортування застосовано в:

- ✅ `ProductDetailsClient.tsx` - сторінка деталей продукту
- ✅ `ProductCard.tsx` - картка продукту в каталозі
- ✅ `ProductQuickViewModal.tsx` - модалка швидкого перегляду
- ✅ `ProductVariantOptions.tsx` - популярні значення розмірів

### 4. Як це працює

```typescript
// Приклад використання
import { sortVariantsBySize } from '@/shared/utils/size-sort';

const variants = [
  { id: '1', options: { size: 'L' } },
  { id: '2', options: { size: 'XS' } },
  { id: '3', options: { size: 'M' } },
];

const sorted = sortVariantsBySize(variants);
// Результат: XS, M, L
```

### 5. Переваги рішення

✅ **Працює без змін бекенду** - сортування на фронтенді
✅ **Автоматичне визначення типу** - літерні або числові розміри
✅ **Підтримка різних форматів** - XS, S, M або 46, 47, 48
✅ **Немає дублювання коду** - одна утиліта для всіх компонентів

### 6. Тестування

Перевірте файл `src/shared/utils/size-sort.test.ts` для прикладів використання.

## Використання

Сортування працює автоматично у всіх компонентах, що відображають варіанти.

При додаванні нових розмірів у адмін-панелі вони автоматично відображатимуться в правильному порядку на фронтенді.
